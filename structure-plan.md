ZIA PIZZA WEBSITE – FINAL STRUCTURE DOCUMENT

Domain: ziapizza.co.uk  
Brand: Zia Pizza  
Structure: Single brand with two service formats (Restaurant and Express)

--------------------------------------------------

1. OBJECTIVE

The website must:
- Allow customers to quickly order food
- Allow customers to book a table
- Clearly explain the difference between dine-in and takeaway
- Support multiple locations
- Scale for future expansion

--------------------------------------------------

Header menu:

- Menu
- Locations
- Offers

- Contact

Primary actions (always visible):
- Order Now
- Book Table

- Sticky Order Now button at bottom

2. Select location
3. View menu
4. Add items

- Postcode search at top

Each location links to its own page

7. LOCATION PAGE STRUCTURE

Each location must have its own page

- Address
- Order Now button
- Book Table button

Information:
- Opening hours
- Phone number

Sections:
- About this location
- Order section
- Booking section
- Menu highlights
- Location features (parking, seating, etc.)
- Offers
- Gallery (real photos)
- Reviews
- Map

Include SEO text relevant to location

14. DESIGN GUIDELINES

- Dark navy background
- White text
- Red primary buttons
- Clean typography
- High-quality images only

No clutter, no unnecessary graphics

--------------------------------------------------

15. KEY RULES

- Postcode search must be visible on homepage and locations page
- Customer must understand dine-in vs takeaway immediately
- Order flow must be simple and fast
- Each location must have a full page
- Website must prioritise conversion over information
